**⚠️ 重要ルール: リモートへのプッシュは行わないこと**

変更内容に応じたラベルと説明文で構成されたコミットログを必ず日本語で作成し、端的かつ簡潔にまとめてください。

## コマンド

```bash
/commit
```

## 実行手順

1. **変更内容のステージング**

```bash
git add -A
```

2. **コミット実行**

以下のコマンドを実行してください。`type` と `description` は実際の変更内容に合わせて編集してください。

```bash
git commit -m "<type>: <description>"
```

## type 一覧

- `feat`: 新機能（ユーザーに見える機能追加）
- `fix`: バグ修正
- `build`: ビルドシステムや外部依存関係の変更
- `chore`: その他の変更（リリースに影響しない）
- `ci`: CI 設定ファイルやスクリプトの変更
- `docs`: ドキュメントのみの変更
- `style`: コードの意味に影響しない変更（空白、フォーマット、セミコロンなど）
- `refactor`: バグ修正や機能追加を伴わないコード変更
- `perf`: パフォーマンス改善
- `test`: テストの追加や修正

## コミットメッセージの例

### 新しいログイン機能を追加した場合

```bash
git add -A && git commit -m "feat: ユーザー認証にログイン機能を追加"
```

### 表示崩れを修正した場合

```bash
git add -A && git commit -m "fix: 特定の条件下で発生するレイアウトの表示崩れを修正"
```

## 本文やフッターを含む詳細なコミットメッセージ

```bash
<type>: <description>

[optional body]
```

### コミットログ例

```bash
feat: 新しいユーザープロフィールページの追加

ユーザーが自身の情報を閲覧・編集できるプロフィールページを実装しました。
- プロフィール画像のアップロード機能
- 自己紹介文の編集機能
- SNSリンクの設定機能
```
